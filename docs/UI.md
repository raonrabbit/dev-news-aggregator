# UI Policy

## Design Direction
- 톤: 미니멀/클린
- 레이아웃: 모바일 우선
- 테마: 라이트/다크 모두 지원
- 원칙: 장식보다 가독성과 정보 전달 우선

## Library Policy
- 기본 정책은 라이브러리 최소 사용입니다.
- Next.js 기본 기능과 경량 스타일(Tailwind 또는 CSS 모듈) 중심으로 구현합니다.
- UI 컴포넌트 라이브러리 도입은 명시 승인 전까지 금지합니다.

## Screen Rules
- 모든 주요 화면은 모바일 기준 레이아웃부터 설계합니다.
- 최소 상태 4종을 구현합니다: loading / empty / error / success.
- 클릭 가능한 요소는 터치 영역을 충분히 확보합니다.
- 타이포는 작은 화면에서도 읽기 쉬운 크기를 유지합니다.

## Accessibility Rules
- 상호작용 요소는 키보드 포커스 이동이 가능해야 합니다.
- 버튼/링크에는 명확한 레이블을 제공합니다.
- 색상만으로 상태를 구분하지 않습니다.
- 라이트/다크 모두에서 텍스트 대비를 확보합니다.

## Dev News Pages UX
- 기사 목록은 스캔 가능한 카드 또는 리스트 형태를 유지합니다.
- 북마크 액션은 즉시 피드백(토스트/상태 변경)을 제공합니다.
- 로그인 유도는 막지 않고 안내하는 방식으로 처리합니다.

## Done Criteria (UI)
- 모바일(좁은 뷰포트)에서 레이아웃 깨짐 없음
- 라이트/다크 전환 시 가독성 유지
- loading/empty/error/success 상태 확인
- 키보드 네비게이션과 기본 접근성 점검 통과
