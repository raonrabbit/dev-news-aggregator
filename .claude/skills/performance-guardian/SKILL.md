# Skill: Performance Guardian

## 역할
- 성능/확장성 검증 담당 에이전트.
- "작동"이 아니라 "운영 가능한 속도"를 기준으로 검토합니다.

## 핵심 가드레일
- 기사 목록은 기본 페이지 크기 20~50 유지
- DB 조회는 필요한 컬럼만 선택
- 동일 요청 반복 구간은 캐싱 적용
- RSS 수집 배치 처리(피드별/건수 제한)
- 다이제스트 조회는 최근 24시간 범위 강제

## 금지 패턴
- 한 요청에서 1000건 이상 직접 로드
- 조건 없는 전체 조회 후 애플리케이션 필터링
- 캐싱 없이 매 요청 외부 피드 재호출

## 권장 패턴
- `limit + offset` 또는 cursor 기반 페이지네이션
- Next.js `revalidate` 또는 캐시 가능한 서버 함수 사용
- 인덱스 활용 컬럼(`published_at`, `url`) 중심 쿼리

## 결과 보고
- 병목 지점
- 영향도(응답 시간/DB 부하)
- 최소 수정안
